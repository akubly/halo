### Specification Locked

1. **Endianness: Little-endian (LE) for all multi-byte fields**
   - BLE ATT layer native (LE).
   - Cortex-M55 (Halo device) native (LE).
   - Python: `struct.pack('<HH', ...)` / `struct.unpack('<H', ...)`
   - Lua: `string.pack('<I2', seq)` / `string.unpack('<I2', data, offset)`

2. **Sequence Number: uint16, wraps 0xFFFF → 0x0000**
   - Host increments monotonically on each `FAMILIAR_UPDATE`; resets at reconnect to 0x0000.
   - Device dedup rule: `delta = (received_seq - last_accepted_seq) mod 65536`; interpret as signed 16-bit:
     - delta 1–32767 = newer (valid, including wraparound) → accept.
     - delta 0 = duplicate → drop silently.
     - delta 32768–65535 (negative) = stale/out-of-order → drop silently.
   - Reconnect reset: device resets `last_accepted_seq` to 0xFFFF; first `0x0000` → delta = 1 → accepted.

3. **Opcode Space**
   - Device→Host: 0x00–0x7F (notifications, ACKs).
   - Host→Device: 0x80–0xFF (commands).

4. **Message Formats**
   - `FAMILIAR_UPDATE` (Host→Device, opcode 0x80): 6 bytes [opcode, mood_enum, intensity, confidence, seq_LE_uint16].
   - `FAMILIAR_ACK` (Device→Host, opcode 0x02): 3 bytes [opcode, last_received_seq_LE_uint16].
   - `FAMILIAR_RESET` (Device→Host, opcode 0x01): 1 byte [opcode] (no payload; occurrence is signal).

5. **FAMILIAR_RESET Direction: Device→Host only**
   - Device snaps to NEUTRAL on double-tap locally (Lua). No `Host→Device` reset opcode exists or should be used.
   - Notification confirms local state to host; host does not command reset.

6. **FAMILIAR_ACK Cadence: Automatic every 10 accepted FAMILIAR_UPDATE packets**
   - ~1 ACK/sec at 10Hz update rate.
   - Unsolicited ACK sent immediately on BLE reconnect, reporting current `last_accepted_seq`.
   - No host-initiated request message defined or needed.

## Decision Rationale

- **Little-endian:** Eliminates byte-order confusion; matches BLE ATT native format and Cortex-M55 architecture.
- **Signed-16 delta dedup:** Correctly handles wraparound without naive `>` comparison that fails at 0xFFFF→0x0000 boundary.
- **Device→Host FAMILIAR_RESET:** Matches Hiro's quick-reset device-ownership decision; no latency burden.
- **Auto ACK cadence:** Simplifies protocol (no request-response handshake); provides periodic ack without host polling.

Specification ensures Python host and Lua device cannot drift on wire format; test fixtures inherit normative spec.
